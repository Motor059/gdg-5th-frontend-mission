# 리액트 폴더 구조 역할 정리

---

## 폴더 구조의 핵심 철학
- 정해진 정답은 없고, **프로젝트의 규모와 복잡도**에 따라 구조를 유연하게 진화시켜야 함

* **`진화 단계`**
    - **초급**: 파일 종류별 분류 (단순함)
    - **중급**: 페이지 단위 분류 (라우팅 중심)
    - **고급**: 기능 단위 분류 (비즈니스 로직 응집도 강화)

* **`목표`**
    - 어떤 파일이 어디에 있는지 한눈에 파악 가능한 **가독성**
    - 기능 수정 시 관련 코드들이 모여 있어 사이드 이펙트를 줄이는 **유지보수성**

---

## 프로젝트 규모별 구조 가이드

### 초급 구조
- 컴포넌트 개수(10~15개 미만)가 적은 토이 프로젝트나 단일 페이지 앱에 적합

* **`구조 특징`**
    - `src` 루트에 `components`, `hooks` 폴더만 존재
    - 모든 컴포넌트를 한 폴더에 넣어 관리하고, 복잡한 계층이 없음

### 2. 중급 구조
- 라우팅이 필요한 일반적인 웹 서비스에 적합하고, **역할별로 폴더를 세분화**함

* **`Components`**
    - 버튼, 인풋 등 재사용 가능한 **UI 요소를 하위 폴더로 분류**하여 관리

* **`Pages`**
    - 라우팅 되는 각 페이지의 진입점을 관리
    - 해당 페이지에서만 쓰이는 로직이나 컴포넌트를 내부에 포함시켜 전역 폴더의 비대화를 막음

* **`Layouts`**
    - 사이드바, 네비게이션 바 등 페이지의 뼈대를 구성하는 컴포넌트 보관

* **`Assets / Context`**
    - **Assets**: 이미지, 폰트, CSS 파일 등 정적 리소스 보관
    - **Context**: 전역 상태 관리를 위한 Context Provider 파일 보관

### 3. 고급 구조
- 대규모 프로젝트에서 **기능(Domain) 간의 결합도를 낮추고 응집도를 높이기 위해** 사용

* **`Features`**
    - 특정 기능(`Todo`, `Auth`, `Payment`)과 관련된 모든 코드(Components, Hooks, Types, API)를 **한 폴더에 뭉쳐 놓음**
    - 다른 기능 개발자는 해당 폴더를 들여다볼 필요가 없어짐

* **`Lib`**
    - 서드파티 라이브러리의 설정을 감싸는 **파사드(Facade) 패턴** 적용
    - `예: axios 인스턴스 생성, dayjs 설정 등`

* **`Services`**
    - 외부 API와 통신하는 비동기 함수들을 모아둠
    - API 코드가 컴포넌트 내부에 산재되지 않고 한곳에서 관리되어 유지보수 용이

* **`Data / Utils`**
    - **Data**: 상수, 환경 변수, 정적 JSON 데이터 등
    - **Utils**: 포매팅 함수, 유효성 검사 등 사이드 이펙트가 없는 **순수 함수**

## Features 폴더와 Public API 패턴
- 기능별로 쪼갠 폴더(`features`)를 외부에서 깔끔하게 사용하기 위한 설계 규칙

* **`캡슐화 (Encapsulation)`**
    - Feature 폴더 내부에는 수많은 파일이 존재하지만, 외부에서는 필요한 것만 접근해야 함

* **`Public API (index.js)`**
    - 각 Feature 폴더의 진입점(`index.js`)에서 **외부에 노출할 컴포넌트나 함수만 `export`** 함
    - 외부에서는 내부 구조를 몰라도 `import { TodoList } from '@/features/todo'` 처럼 깔끔하게 사용 가능

* **`ESLint & Path Alias`**
    - **Path Alias**: `../../features/todo` 대신 `@features/todo` 처럼 절대 경로 설정 권장 (`jsconfig.json` / `tsconfig.json`)
    - **ESLint Rule**: `no-restricted-imports` 규칙을 사용하여 Feature 내부 파일에 직접 접근하는 것을 막고, `index.js`를 통하도록 강제할 수 있음
